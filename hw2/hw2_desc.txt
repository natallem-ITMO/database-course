Опишите выделенные сущносности (в том числе, слабые), их атрибуты.
Groups - группы в университете
	Id - id группы
	GroupName - имя группы

Students - сущность для информации о студенте
	Id - id студента
	FirstName/LastName - имя/фамилия студента
	Course - курс обучения студента

Subjects - предметы в университете
	Id - id 
	SubjectName - название предмета

Marks - оценки университета за предмет
	Mark - собственно, оценка

Professors - преподаватели университета 
	Id - id
	FirstName/SecondName/Patronymic - Имя/Фамилия/Отчество
	Experience - количество лет преподавания



Опишите выделенные связи и ассоциации.
Students-Groups => Каждый студент находится в одной группе. В группе может быть множество стунедтов
Strudents-Subjects => У каждого студента может быть множество предметов. Каждый предмет может посещать множество студентов.
Subjects-Professors => У каждого предмета есть преподаватель, который его преподает. При этом один преподаватель может преподавать несколько предметов.
Grade association => Ассоциация для задания оценки по предмету у заданного студента.
	AssessmentDate - дата выставления оценки (может не быть, если предмет не закрыт)
	ClosingDate - дата закрытия ведомостей по предмету (дедлайн выставления оценки)


id -> int

string -> varchar(255)

mark -> int
years -> int

Опишите особенности преобразования (в частности, для слабых сущностей и ассоциаций).
Students-Groups => Добавлен внешний ключ GroupId у студента на группы
Strudents-Subjects => Добавлена LinkTable StudentSubject для связи Многие ко многим со StudentId и SubjectId внешними ключами.
Subjects-Professors => Добавлены внешние ключи: у профессора SubjectId на выделенный предмет, который он преподает (т.к. профессор должен преподавать хотя бы один предмет), в Subjects - внешний ключ ProfessorId для ссылки на профессора, преподающего предмет.
Grade association => Преобразована в сущность с PK = (StudentId, SubjectId) (они же будут FK), т.к. оценка за предмет определяется парой предмет+студент. Оценки может и не быть (поэтому FK3 - Oblig.). Внешний ключ ссылается на таблицу StudentSubject для того, чтобы студент не получил оценку за предмет, которого у него нет.
